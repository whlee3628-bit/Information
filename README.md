자료 찾기 웹사이트(안암골 서재)

#목적: ‘나는 고려대 학생이고, 과제 하는 고대생을 위한 자료 찾기 웹사이트를 만들고 싶어.’

#기본 PRD 세팅
1. 왜? 자신이 원하는 자료 찾기
2. 누가? 대학생
3. 무엇을? 자료의 주제, 자료의 종류, 자료의 신뢰도, 자료가 작성된 날짜
4. 언제? 대학교 과제 수행(ex. 글쓰기 과목 소논문 쓰기) 
5. 어디서? 웹 브라우저
6. 어떻게? 반응형 웹페이지

동기: 1학년 교양 필수 글쓰기 수업의 소논문 작성을 위해 자료 탐색 사이트 DBPia를 이용했습니다. DBPia는 풍부한 데이터베이스를 갖고 있다는 장점이 있지만 인터페이스가 다소 단조로우며 무엇보다 자료의 출처에 대한 신뢰성 판단을 학생 본인이 해야한다는 점이 아쉬웠습니다. 이에 AI의 직접 평가와 검증을 바탕으로 자료의 신뢰도를 A~D등급으로 나눈 자료 찾기 웹페이지를 만들기로 결심했습니다. 동시에 크림슨색과 호랑이 이미지를 바탕으로 디자인을 조금 더 세련되게 꾸몄습니다.
기능: 검색, 정렬(최신순/오래된순/관련도순), 즐겨찾기, 카테고리 설정, 연도 범위 설정 등
한계: 초기에는 고려대학교 학생들만을 대상으로 한 자료 찾기 웹사이트라는 취지를 살리기 위해 가입 및 로그인 화면을 만드려고 했으나 개인정보 상의 문제와 정보 저장 문제로 인해 포기했습니다. 또한 자료의 종류와 관련하여 초기에는 블로그, 유튜브, 커뮤니티 등의 다양한 출처를 포함하려고 했으나 이렇게 되면 너무 방대한 양의 자료가 검색되는 만큼 학술논문으로 범위를 제한했습니다.
